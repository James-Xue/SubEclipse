# 技术方案三轮评审记录（基于子代理研究结果）

评审人：GitHub Copilot（GPT-5.3-Codex）  
日期：2026-03-11

## 第一轮评审：可行性与边界
### 评审目标
验证方案是否在 Linux 上可落地，并清晰覆盖“框选 ROI + 非文字透明 + 文字遮挡”。

### 发现问题
1. Wayland 能力描述偏技术化，缺少可交付边界定义。
2. 首发路径优先级不够明确，容易导致排期分散。
3. 故障路径未与状态机绑定，恢复策略不完整。

### 修改建议
1. 明确 X11 为首发主路径，Wayland 以受限模式交付。
2. 补充 Wayland L1/L2/L3 能力分级和 UI 提示规范。
3. 定义状态转移：`Running -> Degraded -> Error -> Recovering`。

### 结论
有条件通过。

## 第二轮评审：实现细节与编码可执行性
### 评审目标
确认工程团队可按文档直接分模块编码并联调。

### 发现问题
1. 参数配置较分散，缺少统一默认参数基线。
2. 检测稳定性未形成“迟滞 + 生命周期 + 平滑”完整闭环。
3. X11 实现关键点（点击穿透、DPI 映射）需写到具体实现层。
4. 性能目标有总指标，但缺少分阶段预算。

### 修改建议
1. 固化默认参数集（置信度、NMS、IoU、EMA、missed、fps）。
2. 增加 `score_enter/score_keep` 与 `min_life_frames`。
3. 明确 X11 `XShmGetImage`、ARGB Overlay、XFixes/XShape 方案。
4. 拆分预算：Capture / Detect+Track / Render / E2E。

### 结论
有条件通过。

## 第三轮评审：验收闭环与发布可控性
### 评审目标
检查需求、技术、测试、里程碑是否形成完整闭环并可验收。

### 发现问题
1. 里程碑与验收项最初映射不完全一一对应。
2. Wayland 降级触发条件缺少明确判定。
3. 异常恢复后缺少“回稳指标”要求。
4. 长稳与权限回归测试阈值未显式固定。

### 修改建议
1. 补充 MVP/Beta/GA 的交付项到 AC 的映射关系。
2. 定义降级触发条件：授权失败、帧流中断、全局置顶不可用。
3. 新增恢复后 60 秒回稳标准（FPS 与 P95 延迟回归）。
4. 固化 4 小时长稳、权限撤销/恢复、多 DPI 回归用例阈值。

### 结论
通过。
