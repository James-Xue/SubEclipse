# 字幕遮挡器（SubEclipse）需求与技术方案（C++）

版本：v1.0  
日期：2026-03-11  
状态：评审通过（3/3）

---

## 0. 背景与目标

根据需求：
1. 用户可框选屏幕区域（ROI）；
2. ROI 内非文字内容保持透明可见；
3. ROI 内文字内容被遮挡。

本方案定位为 Linux 桌面优先、C++20 实现，首版优先保证“可用、低延迟、可维护”。

---

## 1. 需求文档

### 1.1 术语定义
- ROI：用户框选的屏幕矩形区域。
- Overlay：透明置顶窗口，仅绘制遮挡图层。
- Text Box：文本检测产出的矩形框。
- Track：跨帧关联后的文本框轨迹。

### 1.2 功能需求（FR）
- FR-01：支持鼠标框选 ROI（新建、移动、缩放）。
- FR-02：支持实时采集 ROI 图像（默认 20 FPS，可配 15–30）。
- FR-03：支持文本检测，输出 bbox 与置信度。
- FR-04：支持文本框追踪与平滑，减少抖动与闪烁。
- FR-05：仅对文本框区域渲染遮挡，其余区域透明。
- FR-06：遮挡样式支持黑框/模糊/像素化。
- FR-07：支持热键（开始/暂停/重选 ROI）。
- FR-08：支持配置持久化。

### 1.3 非功能需求（NFR）
- NFR-01：MVP 端到端延迟 P95 ≤ 120ms；Beta 目标 ≤ 80ms。
- NFR-02：1080p 字幕条 ROI 下 CPU 平均 < 35%（4c/8t）。
- NFR-03：连续运行 4 小时稳定无崩溃，内存增长 < 10%。
- NFR-04：默认本地处理，不上传图像帧。

### 1.4 边界与约束
- 不修改底层视频源像素，仅做前景 overlay。
- 不保证 OCR 内容识别绝对正确，首要目标是文本区域遮挡。
- Wayland 下存在全局置顶与自由覆盖限制，需提供降级模式。

## 2. 技术方案

### 2.1 总体架构
- 控制层：`AppController`（状态机、线程编排、生命周期）。
- 平台层：`IScreenCapture`（X11 / Wayland Portal 实现）。
- 视觉层：预处理 + 文本检测 + 追踪平滑。
- 渲染层：`OverlayRenderer`（透明窗口 + 文字区域遮挡）。
- 配置与指标层：配置读写、FPS/延迟/资源统计。

### 2.2 数据流
`ROI` → `Capture` → `Detect` → `Track` → `Render`。

### 2.3 线程模型（4 线程）
- UI 主线程：窗口事件、热键、参数更新。
- Capture 线程：固定节拍抓取 ROI。
- Vision 线程：检测 + 跟踪（仅处理最新帧，丢弃过时帧）。
- Render 线程：刷新 overlay。

并发建议：
- `capture -> vision`、`vision -> render` 采用 SPSC 队列；
- 环形缓冲区深度默认 3；
- 以低延迟优先，不追求全帧处理。

### 2.4 平台可行性

#### 2.4.1 X11（首版推荐）
- 抓屏：`XShmGetImage`（低延迟），按 ROI 裁剪。
- Overlay：ARGB 透明窗口 + 置顶 + 输入穿透（XFixes/XShape）。
- 可满足“全局覆盖 + ROI 遮挡”核心目标。

#### 2.4.2 Wayland（受限支持）
- 抓屏需 `xdg-desktop-portal` + PipeWire 用户授权。
- 全局置顶覆盖受 compositor 策略约束，不保证一致行为。
- 首版建议：
  - 模式 A：Portal 抓屏 + 应用窗口内预览遮挡（通用）；
  - 模式 B：wlroots 环境可探索 layer-shell 增强（非通用）。

### 2.5 核心算法（透明非文字、遮挡文字）

默认参数：
- `fps=20`
- `conf_threshold=0.45`
- `nms_iou=0.30`
- `match_iou=0.40`
- `ema_alpha=0.65`
- `min_text_height=12`
- `full_detect_interval=3`
- `max_missed_frames=6`

伪代码：
```cpp
for each frame_tick:
    frame = capture_roi()
    if frame.empty(): continue

    if need_full_detect(frame_id, interval=3):
        det = detect_text(frame, conf=0.45, nms=0.30)
    else:
        det = {}

    tracker.predict(dt)
    tracker.update_with_iou(det, 0.40)
    boxes = tracker.stable_boxes(ema_alpha=0.65,
                                 min_h=12,
                                 max_missed=6)

    renderer.render_mask(boxes, style)
```

遮挡样式：
- 黑框：性能最高，默认；
- 模糊：视觉自然，CPU 开销中等；
- 像素化：性能与效果折中。

### 2.6 接口草案（C++20）
```cpp
struct Rect { int x, y, w, h; };
struct TextBox { Rect rect; float score; };
struct Frame { int64_t ts_ms; int width, height; std::vector<uint8_t> rgba; };

class IScreenCapture {
public:
  virtual ~IScreenCapture() = default;
  virtual bool init() = 0;
  virtual bool setRoi(const Rect& roi) = 0;
  virtual bool grab(Frame& out) = 0;
};

class ITextDetector {
public:
  virtual ~ITextDetector() = default;
  virtual bool loadModel(const std::string& modelPath) = 0;
  virtual std::vector<TextBox> detect(const Frame& frame) = 0;
};

class TextTracker {
public:
  void predict(double dtMs);
  void update(const std::vector<TextBox>& detections);
  std::vector<Rect> stableBoxes() const;
};

class OverlayRenderer {
public:
  bool createWindow();
  void setStyle(int style, int blurKernel, int pixelBlock);
  void render(const std::vector<Rect>& boxes);
};
```

### 2.7 推荐技术栈（首版）
- GUI/窗口：Qt 6
- 图像处理：OpenCV 4.x
- 文本检测：PaddleOCR 检测模型（ONNX Runtime）
- 构建：CMake

推荐理由：工程成熟、社区资料足、跨平台能力好、迭代快。

## 3. 实施计划与验收

### 3.1 里程碑
- MVP（2–4 周）
  - X11：ROI 框选 + 文本遮挡（黑框）+ 参数配置。
- Beta（3–5 周）
  - 加入追踪平滑 + 模糊/像素化 + 性能指标 + Wayland 受限模式。
- GA（3–4 周）
  - 兼容性完善、打包发布、日志诊断工具。

### 3.2 验收标准
- 功能验收：
  - 能稳定框选 ROI；
  - ROI 内非文字透明可见；
  - ROI 内文字可被持续遮挡（不明显闪烁）。
- 性能验收：
  - 1080p 字幕条 ROI 下平均 20 FPS；
  - 端到端 P95 延迟 ≤ 120ms（MVP）。
- 稳定性验收：
  - 连续运行 4 小时无崩溃。

### 3.3 测试计划
- 单元测试：IoU/NMS、EMA 平滑、配置加载。
- 集成测试：抓屏→检测→渲染链路。
- 性能测试：固定视频集回放，统计 P50/P95。
- 兼容测试：多发行版、多显示器、DPI 100/125/150%。

### 3.4 风险与缓解
- OCR 漏检：检测与追踪融合 + 迟滞阈值。
- 动态字幕抖动：Kalman/EMA 与最小生命周期过滤。
- Wayland 限制：能力探测 + 降级模式 + 明确用户提示。
