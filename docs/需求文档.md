# 字幕遮挡器需求文档（SubEclipse）

版本：v1.1
日期：2026-03-11

## 1. 背景与目标
本项目面向 Linux 桌面，实现字幕区域的实时遮挡：
- 用户可框选屏幕 ROI（字幕区域）；
- ROI 内非文字区域保持透明可见；
- ROI 内文字区域被遮挡。

## 2. 术语定义
- ROI：用户框选的屏幕矩形区域。
- Overlay：透明置顶渲染层，仅绘制遮挡内容。
- Text Box：文本检测输出框（矩形 + 置信度）。
- Track：跨帧关联后的稳定文本框轨迹。
- 端到端延迟：帧采集到遮挡显示的时间差。

## 3. 功能需求（FR）
- FR-01：支持 ROI 新建、移动、缩放、重置。
- FR-02：支持开始/暂停/继续遮挡流程。
- FR-03：支持 ROI 内实时文本区域检测。
- FR-04：仅遮挡文本区域，非文本区域透明显示。
- FR-05：支持遮挡样式切换（黑框/模糊/像素化）。
- FR-06：支持文本框跨帧稳定（抖动抑制）。
- FR-07：支持热键（开始、暂停、重选 ROI、退出）。
- FR-08：支持配置持久化（参数、样式、热键、ROI）。
- FR-09：X11 下支持完整能力路径。
- FR-10：Wayland 下支持受限模式并明确提示能力边界。
- FR-11：显示运行指标（FPS、延迟、CPU、掉帧）。
- FR-12：关键故障可提示并可恢复（模型失败/抓屏中断/权限撤销）。

## 4. 非功能需求（NFR）
- NFR-01：MVP 延迟 P95 ≤ 120ms，P50 ≤ 70ms。
- NFR-02：Beta 延迟 P95 ≤ 80ms，P50 ≤ 45ms。
- NFR-03：1080p 字幕条 ROI 下平均 FPS ≥ 20，P95 FPS ≥ 18。
- NFR-04：4c/8t 设备下平均 CPU ≤ 35%，P95 CPU ≤ 50%。
- NFR-05：常驻内存 ≤ 500MB，4 小时增长 ≤ 10%。
- NFR-06：冷启动到可操作 ≤ 3s（模型已缓存）。
- NFR-07：默认本地处理，不上传帧与识别结果。
- NFR-08：崩溃后可恢复最近配置。
- NFR-09：多 DPI 下 ROI 与遮挡坐标误差 ≤ 2px。
- NFR-10：抓屏中断自动恢复时间 ≤ 2s。

## 5. 边界与不做项
- 不输出 OCR 文字内容，不做翻译。
- 不修改视频源像素，仅做前景 Overlay。
- 不承诺 100% 文本召回率，优先低延迟与稳定。
- 不承诺 Wayland 全环境具备统一全局置顶能力。
- 不做云端推理、账号系统、远程同步。

## 6. 验收标准（AC）
- AC-01：ROI 交互成功率 ≥ 99%。
- AC-02：ROI 与渲染坐标偏差 ≤ 2px（100/125/150% DPI）。
- AC-03：文本遮挡命中率 ≥ 92%（IoU ≥ 0.5）。
- AC-04：非文本误遮挡面积占比 ≤ 8%。
- AC-05：遮挡框抖动幅度 P95 ≤ 3px。
- AC-06：闪烁率 ≤ 3 次/分钟。
- AC-07：满足 NFR-01、NFR-03、NFR-04。
- AC-08：连续 4 小时无崩溃并满足 NFR-05。
- AC-09：X11 完成 FR-01~FR-12 全量验收。
- AC-10：Wayland 完成受限模式：授权采集、局部遮挡、能力提示。
- AC-11：模型失败、抓屏中断、权限撤销可恢复或降级。
- AC-12：日志可导出采集/检测/跟踪/渲染耗时统计。
