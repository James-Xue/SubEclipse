# 字幕遮挡器技术方案（SubEclipse）

版本：v1.1
日期：2026-03-11

## 1. 分层架构
- App Layer：状态机、生命周期、配置、恢复策略。
- Platform Layer：抓屏、窗口、输入穿透、DPI 映射。
- Vision Layer：预处理、文本检测、跟踪与平滑。
- Render Layer：遮挡生成与 Overlay 绘制。
- Observe Layer：性能指标、日志、诊断导出。

## 2. 模块职责
- `AppController`：状态机（Idle/Selecting/Running/Paused/Degraded/Error）。
- `RoiManager`：ROI 交互、边界约束、坐标转换。
- `CaptureService`：定时采集 ROI 帧并打时间戳。
- `TextDetector`：模型加载、推理、后处理。
- `TrackManager`：关联、生命周期管理、抖动抑制。
- `MaskComposer`：稳定框转遮挡绘制指令。
- `OverlayRenderer`：透明窗口渲染、样式控制、点击穿透。
- `MetricsCollector`：FPS、P50/P95 延迟、CPU/RSS、掉帧率。

## 3. 线程与时序
### 3.1 线程模型
- UI 主线程：事件、热键、参数热更新。
- Capture 线程：固定节拍采集。
- Vision 线程：检测 + 跟踪（仅消费最新帧）。
- Render 线程：绘制遮挡层。

并发策略：
- SPSC 队列：`captureQueue`（3）、`maskQueue`（2）。
- 队列满时丢旧保新，低延迟优先。
- 参数通过原子快照更新：`std::atomic<std::shared_ptr<RuntimeConfig>>`。

### 3.2 时序
1. 用户框选 ROI。
2. Capture 每 50ms 抓取 ROI 帧。
3. Vision 执行检测/关联/平滑。
4. Render 绘制文本框遮挡。
5. Metrics 记录阶段耗时。

## 4. 核心算法实现细节
### 4.1 检测
- 输入：ROI RGBA/BGR。
- 预处理：短边缩放到 640，保持纵横比。
- 推理：ONNX Runtime（CPU EP）。
- 后处理：阈值过滤 + NMS。

### 4.2 跟踪
- 关联条件：IoU + 中心距离双门限。
- MVP 可用贪心 IoU，后续可切匈牙利算法。
- 生命周期：`spawn/update/missed/prune`。

### 4.3 迟滞与抖动抑制
- 进入阈值：`score_enter=0.52`。
- 保持阈值：`score_keep=0.42`。
- 生存门限：`min_life_frames=2`，`max_missed_frames=6`。
- 平滑：EMA，`alpha=0.65`（位置与尺寸都平滑）。

### 4.4 遮挡渲染
- 仅绘制文本框区域，不填充非文本区域。
- 默认黑框：`RGBA(0,0,0,220)`。
- 框扩展：`pad_x=3`、`pad_y=2`。
- 样式：黑框 / 模糊（kernel=9）/ 像素化（block=8）。
- 合并：IoU > 0.6 的重叠框合并，减少过绘与闪烁。

## 6. C++ 接口草案
```cpp
enum class BackendType { X11, WaylandPortal };
enum class MaskStyle { SolidBlack, Blur, Pixelate };
enum class AppState { Idle, SelectingROI, Running, Paused, Degraded, Error };

struct RectI { int x, y, w, h; };
struct Frame {
  uint64_t frame_id;
  int64_t ts_capture_ms;
  int width;
  int height;
  std::vector<uint8_t> rgba;
};

struct TextDetection { RectI rect; float score; };
struct StableBox { RectI rect; float score; uint32_t track_id; };

class IScreenCapture {
public:
  virtual ~IScreenCapture() = default;
  virtual bool init() = 0;
  virtual bool set_roi(const RectI& roi_screen_px) = 0;
  virtual bool grab(Frame& out) = 0;
  virtual void shutdown() = 0;
};

class ITextDetector {
public:
  virtual ~ITextDetector() = default;
  virtual bool load_model(const std::string& model_dir) = 0;
  virtual std::vector<TextDetection> detect(const Frame& frame) = 0;
};

class IOverlayRenderer {
public:
  virtual ~IOverlayRenderer() = default;
  virtual bool create_overlay() = 0;
  virtual bool set_roi(const RectI& roi_screen_px) = 0;
  virtual void set_style(MaskStyle style) = 0;
  virtual void render(const std::vector<StableBox>& boxes) = 0;
  virtual void set_click_through(bool enabled) = 0;
  virtual void shutdown() = 0;
};
```

## 7. Linux 平台实现
### 7.1 X11（主路径）
- 抓屏：优先 `XShmGetImage`，失败回退 `XGetImage`。
- Overlay：ARGB Visual + 置顶属性（`_NET_WM_STATE_ABOVE`）。
- 点击穿透：XFixes/XShape 设置 Input Shape 为空。
- ROI 编辑模式临时取消穿透。
- 坐标统一为物理像素，UI 层做逻辑像素映射。

### 7.2 Wayland（受限模式）
- 抓屏：`xdg-desktop-portal` + PipeWire。
- 降级策略：
  - L1：可抓屏 + 应用内遮挡预览；
  - L2：可抓屏但不可可靠全局置顶；
  - L3：授权失败，仅保留配置与提示。

## 8. 性能预算与可观测性
### 8.1 预算（1080p 字幕条 ROI）
- Capture：P50 8ms / P95 14ms
- Detect+Track：P50 22ms / P95 45ms
- Render：P50 4ms / P95 8ms
- 端到端：P50 ≤ 45ms，P95 ≤ 80ms（Beta）

### 8.2 观测指标
- 帧级时间戳：`ts_capture`、`ts_detect_done`、`ts_render_done`。
- 统计：`fps_current`、`latency_p50/p95`、掉帧率。
- 资源：CPU%、RSS、线程占用。

## 9. 容错与恢复
- 模型加载失败：进入 `Degraded`，允许重试/切换模型。
- 抓屏中断：指数退避重连（100ms~2s）。
- 权限撤销：停采集并提示重新授权。
- 渲染失败：回退黑框 + 降低刷新率 + 重建窗口。
- 配置损坏：自动回退默认配置并备份坏文件。

## 11. CMake 实施方案（首版）
### 11.1 最小目录与构建文件
- 顶层：`CMakeLists.txt`
- 构建辅助：`cmake/Dependencies.cmake`
- 源码：`src/`
- 头文件：`include/`
- 测试：`tests/`

### 11.2 顶层 CMake 目标
- `subeclipse_core`：核心库（app/vision/render/metrics 公共逻辑）。
- `subeclipse_platform_x11`：X11 平台库（抓屏 + overlay）。
- `subeclipse_app`：可执行程序（链接 core + x11）。
- `subeclipse_tests`：测试目标（可选，MVP 可后置）。

### 11.3 推荐 CMake 关键项
- `cmake_minimum_required(VERSION 3.22)`
- `project(SubEclipse VERSION 0.1.0 LANGUAGES CXX)`
- `set(CMAKE_CXX_STANDARD 20)`
- `set(CMAKE_CXX_STANDARD_REQUIRED ON)`
- `set(CMAKE_EXPORT_COMPILE_COMMANDS ON)`
- 构建类型：默认 `RelWithDebInfo`。

### 11.4 依赖发现与链接建议
- 必需：`Threads`、`X11`、`Xext`、`Xfixes`、`Xrender`、`Xcomposite`。
- 视觉链路：`OpenCV`（MVP 可先只用 core/imgproc）。
- 推理链路：`ONNXRuntime`（可通过 `find_package` 或 imported target）。
- GUI 层（如采用 Qt）：`Qt6::Core`、`Qt6::Gui`、`Qt6::Widgets`。

### 11.5 编译选项建议
- Debug：`-O0 -g3 -fno-omit-frame-pointer`
- Release：`-O3 -DNDEBUG`
- 通用告警：`-Wall -Wextra -Wpedantic`
- 可选：`-Werror`（CI 启用，本地可关闭）

### 11.6 首版构建命令
- 配置：`cmake -S . -B build -DCMAKE_BUILD_TYPE=RelWithDebInfo`
- 编译：`cmake --build build -j`
- 运行：`./build/subeclipse_app`

### 11.7 与周计划的映射
- 第 1 周必须完成：
  - 构建脚手架 + 最小可执行程序；
  - `subeclipse_core` 与 `subeclipse_platform_x11` 两个目标可独立编译。
- 详细执行步骤见：`docs/每周可执行清单.md`。
